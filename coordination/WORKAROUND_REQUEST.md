## Request Template

When Dev Agent needs a workaround, they write:

```markdown
## Workaround Request: [Brief Title]

**Date**: [timestamp]
**Agent**: Dev
**Task**: TASK-XXX

### The Problem
[What you're trying to implement]

### Why Exact Solution is Difficult
[Technical reasons - be specific]

### Proposed Workaround
[What you would do instead]

### What We Lose
[Trade-offs of the workaround]

### Effort Comparison
- Exact solution: [estimated effort]
- Workaround: [estimated effort]

### Recommendation
[Your recommendation, but user decides]
```
