
**Duration:** 20–30 min | **Participants:** 15–25 students  
**Format:** Hands-on, think-aloud optional  
**Test URL:** [AI-Visual Playground](https://source-academy.github.io/pie-slang/tree/ai-visual)

---

## Part 1: First Impressions (3–5 min)

Open the playground. Do **not** click yet. Just look.  

**Questions:**
1. What do you think this tool does?  
2. Describe the main sections/panels.  
3. Where would you type code? Where would output appear?  
4. Do you notice AI features or visual aids?  
5. First impression: Modern / Cluttered / Minimalist / Overwhelming?  

---

## Part 2: Basic Interactions (10 min)

**Task 1: Run Code**
- Type `(add1 zero)` in the editor
- Click "Run"
- **Feedback:**  
  - Ease finding the run button (1–5)  
  - Output visibility & clarity (1–5)  

**Task 2: Load an Example**
- Find the "Examples" menu, select one
- **Feedback:**  
  - Ease of finding & using examples (1–5)  
  - Does code replace existing code or append?  

**Task 3: Introduce an Error**
- Type `(+ 1 "hello")` and run
- **Feedback:**  
  - Error visibility & clarity (1–5)  
  - Any AI suggestions?  

**Optional Task 4: Explore Visual/AI Features**
- Click any "Visualize" or "AI help" buttons
- **Feedback:**  
  - Usefulness (1–5)  
  - Confusing or clear?  

> **Tip:** Students don’t need to explore everything — just what catches their eye.

---

## Part 3: AI & Visualization Features (5–7 min)

- Try AI suggestions (if available)  
- Load an example with proof or type info  
- Hover to see type info  
- Look for execution traces / visualization  

**Quick Feedback:**  
- AI discoverable & usable? (1–5)  
- Visualizations clear & helpful? (1–5)  
- Overall impression of AI integration (1–5)  

---

## Part 4: Layout & Design (3–5 min)

**Questions:**
1. How flexible is the layout? Can panels be resized/hidden?  
2. Is visual hierarchy clear? Editor main focus?  
3. Colors/syntax highlighting intuitive?  
4. Overall visual design (1–5)
